# Phase 2 수동 테스트 체크리스트

Phase 2에서 구현된 모든 기능에 대한 수동 테스트 가이드입니다.

## 테스트 환경 준비

```bash
# Docker Compose로 전체 서비스 실행
docker-compose up --build

# 서비스 접근
- Frontend: http://localhost:5173
- Backend API: http://localhost:3000
- PostgreSQL: localhost:5432
```

## 1. Authentication UI 테스트 (Step 1)

- [ ] 필수 항목 미입력 시 에러 메시지 표시
- [ ] 회원가입 성공 시 대시보드로 자동 리다이렉트
- [ ] 회원가입 성공 시 토스트 알림 표시

### 로그인
- [ ] `/login` 페이지 접근 가능
- [ ] 이메일, 비밀번호 입력 가능

### Premium Access Model

### 챕터 페이지 - 퀴즈 섹션

- [ ] 객관식 문제: 선택지 라디오 버튼 동작
- [ ] O/X 문제: 참/거짓 라디오 버튼 동작
- [ ] 단답형 문제: 텍스트 입력 필드 동작
- [ ] 각 문제의 배점 표시

## 자동화 테스트 (향후 계획)

현재는 수동 테스트로 진행하지만, 향후 다음 자동화 테스트를 추가할 예정입니다:

### Unit Tests
- API Client 함수 테스트
- Zustand Store 테스트
- Utility 함수 테스트

### Integration Tests
- 컴포넌트 렌더링 테스트 (React Testing Library)
- 사용자 인터랙션 테스트
- API 통신 테스트 (MSW)

### E2E Tests
- Playwright 또는 Cypress를 사용한 전체 플로우 테스트
- 주요 사용자 시나리오 자동화
